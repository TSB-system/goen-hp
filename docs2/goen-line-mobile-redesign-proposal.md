# 語縁 GOEN モバイルLP改善提案

## 1. 全体のデザインコンセプト

「静かな信頼感の中に、導線が一本通っているモバイルLP」。
生成り、墨黒、深緑、控えめな金色を基調に、余白、細い罫線、明朝体、導線を感じる縦のリズムで「見つかる、信頼される、相談される」を自然に伝える。

## 2. 現状デザインの課題まとめ

- ファーストビューで写真、斜めライン、コピー、CTAの関係が弱く、サービス内容がやや伝わりにくい。
- セクションが縦に長く、同じリズムが続いて読み疲れしやすい。
- 背景英字の存在感が強い箇所があり、本文の主役感が弱まる。
- 価格、FAQ、相談導線の不安解消が少なく、CV前の安心材料が不足している。

## 3. 改善後のサイト全体の印象

上品で静かな世界観を保ちながら、「何をしてくれるか」「自分に関係あるか」「いくらか」「相談しても大丈夫か」が短いスクロールで理解できる構成へ変更。
カードは安っぽくせず、生成りのトーン差と細い罫線で情報を整理する。

## 4. ファーストビュー

・現状の課題
背景写真が暗く、手元と文字の関係が弱い。斜め線が装飾に見え、CTAまでの視線誘導が散る。

・改善方針
深緑から墨黒の自然なグラデーションで写真をなじませる。斜め線は1〜2本に抑え、「一本の線」の象徴にする。

・改善後の見え方
ロゴ、対象者、メインコピー、安心材料、CTAが上から自然に読める。

・コピー修正案
地域店舗・個人事業主のための Web導線設計
見つかる。信頼される。相談される。
HP・LP・MEO・LINE・SNSまで、必要な施策だけを一本の線で整えます。

・CTA導線
主CTA「無料で相談する」、サブCTA「導線設計の考え方を見る」。

## 5. セクション別改善案

### QUALITY

・現状の課題: 長文から始まり、価値が整理されて見えにくい。
・改善方針: 「見つけられる / 信頼される / 相談される」を3要素で先に提示。
・見え方: 価値カードの後に本文を置き、読む前に全体像が分かる。
・CTA導線: 直接CTAは置かず、SYSTEMへ心理的に接続する。

### PROBLEM

・現状の課題: 悩みは伝わるが、単調な縦長に見える。
・改善方針: 3カードに整理し、番号、英字、見出し、説明の階層を明確にする。
・見え方: 煽らず、共感として「選ばれない理由」が分かる。
・CTA導線: ここでは押さず、解決思想のSYSTEMへ進ませる。

### SYSTEM

・現状の課題: 6ステップが長く見えやすい。
・改善方針: 細い縦線とコンパクトなステップで「線」を見せる。
・見え方: 集客を点ではなく流れとして理解できる。
・CTA導線: 解決思想を納得させ、PRICEへ進む。

### PRICE

・現状の課題: 価格の目安はあるが、相談前の不安解消が弱い。
・改善方針: 2列カード化し、目的と価格を比較しやすくする。直後に安心コピーとCTAを置く。
・CTA: 「料金について無料で相談する」。

### SERVICE

・現状の課題: サービス名が先に立ち、何が変わるかが遅れて伝わる。
・改善方針: 「信頼をつくる」「問い合わせを増やす」など用途を先に出す。
・見え方: 相談前の人が自分の課題に合わせて見られる。

### WORKS

・現状の課題: サンプル画像が装飾に寄り、導線設計の具体性が弱い。
・改善方針: Googleマップ → LP/HP → LINE・フォームを細い線で表現。
・見え方: LP、MEO、LINEが連動して成果導線になると分かる。

### INSIGHT

・現状の課題: 文章量が多く、黒背景で重く見えやすい。
・改善方針: 実体験に基づく設計力として短い段落で表現する。
・見え方: 押しつけがましくない信頼感が出る。

### PLANS

・現状の課題: プラン一覧に見え、相談の入口感が弱い。
・改善方針: 4つの「はじまり方」としてカード化し、各カードに相談導線を置く。
・CTA: 「この内容で相談する」「自分に合う始め方を相談する」。

### FLOW

・現状の課題: 長いタイムラインに見えやすい。
・改善方針: 契約前の安心感を出し、説明を1〜2行へ短縮。
・見え方: 初回相談から制作・改善まで不安なく理解できる。

### FAQ

・現状の課題: 押しやすさはあるが、CTAへの接続が弱い。
・改善方針: アコーディオン直後に安心コピーとCTAを追加する。
・CTA: 「不安なことも含めて無料で相談する」。

## 6. 改善後コピー案

- 見つかる。信頼される。相談される。
- 見た目だけで終わらない。相談につながる導線を設計します。
- 頑張っているのに、選ばれない。足りないのは努力ではなく、導線です。
- 集客は、点ではなく線で考える。
- 料金の不安を、最初に解消する。
- 必要なところから、必要な分だけ。
- あなたに近い、はじまり方は？
- 不安なことも含めて、最初に整理できます。

## 7. レイアウト指示

- モバイル専用ページとして最大幅430pxで設計し、広い画面では中央にスマホ幅で表示する。
- セクションは `76〜112px` 程度の上下余白で、情報の密度を保つ。
- カードは角丸を使わず、薄い生成り背景と細い罫線で区切る。
- CTAは幅100%、高さ58px、深緑背景、薄い金色枠、白文字。

## 8. 具体ルール

- ベース: `#F4EFE5`、濃い生成り: `#EDE5D8`
- 墨黒: `#1C1A16`、深緑: `#0F3D2E` / `#114832`
- 金色: `#B89A5B`、薄金: `#D2BE8A`
- 本文: 14.5〜16px、行間1.85〜2.0
- 見出し: 29〜38px、行間1.45
- 背景英字は `-webkit-text-stroke` で薄く表示し、本文より主張させない。

## 9. 問い合わせ率を上げる導線改善案

1. Heroで対象者と提供価値を即理解させる。
2. QUALITYで価値を3つに整理する。
3. PROBLEMで自分ごと化する。
4. SYSTEMで解決思想に納得させる。
5. PRICEで料金不安を下げ、CTAを置く。
6. WORKSとINSIGHTで信頼を補強する。
7. PLANSで自分に近い入口を選ばせる。
8. FAQで営業不安を解消し、CONTACTへ進ませる。

## 10. 改善後のモバイルワイヤーフレーム

Hero → QUALITY → PROBLEM → SYSTEM → PRICE + CTA → SERVICE → WORKS → INSIGHT → PLANS + CTA → FLOW → FAQ + CTA → ABOUT → CONTACT

## 11. 修正優先順位

1. Heroの自然化とコピー階層整理
2. PRICE、PLANS、FAQのCTA強化
3. QUALITY、PROBLEM、SERVICEのカード整理
4. WORKSの導線表現
5. INSIGHTの文章量調整

## 12. 実装時の注意点

- Web版HTMLには触れず、モバイル専用HTML/CSS/JSで完結させる。
- 既存画像を使い、追加素材が必要になるまで新規画像は増やさない。
- 装飾を増やすより、余白、罫線、コピーの階層で上品さを出す。
- CTAは目立たせるが、派手な影や強いグラデーションは避ける。
